# 缩放范围更新 - 2025-04-01

## 更新内容

将精细Map修订工具的缩放范围从 **0.1x ~ 10.0x** 扩展到 **0.1x ~ 50.0x**

## 修改原因

用户反馈最大缩放倍数只能达到9.85x,不够用。需要更大的缩放倍数以便更精细地查看和编辑Map点位。

## 修改内容

### 代码修改
**文件**: `ui_fine_map_revision.py`
**位置**: 第249行,`wheelEvent`方法

**修改前**:
```python
# 限制缩放范围
if 0.1 <= new_scale <= 10.0:
    self.scale_factor = new_scale
```

**修改后**:
```python
# 限制缩放范围 (0.1x 到 50.0x)
if 0.1 <= new_scale <= 50.0:
    self.scale_factor = new_scale
```

## 实际效果

### 修改前
- 最大只能放大到10倍
- 对于间距小的Map(如1mm间距),点位显示较密集
- 难以精确选择和编辑相邻点位

### 修改后
- 可以放大到50倍
- 相同间距的Map点位显示更清晰
- 可以更精确地选择和编辑单个点位
- 适合高精度Map的精细编辑

## 使用说明

### 缩放操作
1. 在可视化图中,使用**鼠标滚轮**进行缩放
2. **向上滚动**: 放大视图
3. **向下滚动**: 缩小视图

### 缩放范围
- **最小**: 0.1x (可以看到整个Map的全景)
- **最大**: 50.0x (可以精确看到每个点位)

### 缩放提示
- 右下角会显示当前缩放倍数
- 例如: "缩放: 12.50x"

## 技术细节

### 缩放算法
```python
# 每次滚轮缩放倍率
zoom_factor = 1.1  # 放大
zoom_factor = 0.9  # 缩小

# 从0.1x到50.0x需要的滚动次数
# 向上滚动: log(50/1)/log(1.1) ≈ 41次
# 向下滚动: log(1/0.1)/log(0.9) ≈ 22次
```

### 坐标变换
缩放时保持鼠标位置不变:
```python
world_pos_after = self.screen_to_world(mouse_pos.x(), mouse_pos.y())
self.offset_x += (world_pos_after[0] - world_pos_before[0]) * self.scale_factor
self.offset_y -= (world_pos_after[1] - world_pos_before[1]) * self.scale_factor
```

## 性能考虑

### 内存使用
- **50x缩放**: 内存使用与10x基本相同
- 渲染点数不变,只是显示放大

### 渲染性能
- **QPainter**: 抗锯齿渲染,50x时性能良好
- **可见区域**: 实际只渲染可见区域的点
- **优化**: 已有视锥剔除(不渲染屏幕外的点)

### 建议
- 对于超大型Map(>10000点),建议不要长时间保持50x
- 正常使用情况下,性能应该没问题
