# 精细Map修订工具 - 使用指南

## 功能概述

精细Map修订工具(Fine Map Revision)是一个全新的功能模块,提供了Map数据的可视化编辑和精细修改能力。

## 核心功能

### 1. 数据管理

#### 导入现有Map
- **菜单路径**: 数据 → 导入现有Map (或 Ctrl+I)
- **支持格式**: CSV文件,包含X和Y两列坐标
- **示例格式**:
  ```csv
  X,Y
  -70,-70
  -68,-70
  -66,-70
  ```

#### 新建Map
- **菜单路径**: 数据 → 新建Map (或 Ctrl+N)
- **参数说明**:
  - X方向长度: Map在X方向的总长度 (mm)
  - Y方向长度: Map在Y方向的总长度 (mm)
  - X方向间距: 相邻两点在X方向的距离 (mm)
  - Y方向间距: 相邻两点在Y方向的距离 (mm)
  - 中心点X坐标: Map中心点的X坐标 (mm)
  -中心点Y坐标: Map中心点的Y坐标 (mm)

#### 保存到CSV
- **菜单路径**: 数据 → 保存到CSV (或 Ctrl+S)
- **输出格式**: 标准CSV格式,包含索引、X坐标、Y坐标

### 2. 交互式可视化

#### 视图操作
- **缩放**: 鼠标滚轮
  - 向上滚动: 放大
  - 向下滚动: 缩小
  - 缩放范围: 0.1x ~ 50.0x

- **平移**: 鼠标左键拖拽
  - 点击并拖动空白区域可平移视图

- **重置视图**: 点击"重置视图"按钮
  - 恢复到初始缩放和位置

#### 点位选择
- **单击点**: 选中该点
  - 选中的点显示为红色
  - 鼠标悬停的点显示为橙色
  - 普通点显示为蓝色

- **双击点**: 弹出编辑对话框
  - 可直接修改该点的X、Y坐标

- **右键菜单**:
  - 添加点: 在鼠标位置添加新点
  - 删除选中点: 删除当前选中的点
  - 重置视图: 恢复视图到初始状态

### 3. 数据表格编辑

#### 表格特性
- **列结构**:
  - 索引: 点的序号(只读)
  - X坐标: 可编辑
  - Y坐标: 可编辑

#### 编辑操作
- **直接编辑**:
  - 双击单元格进入编辑模式
  - 选中单元格后按F2键进入编辑模式

- **选择同步**:
  - 在表格中选择行 → 可视化图中对应点被选中
  - 在可视化图中选择点 → 表格中对应行被选中

#### Excel兼容操作
- **Ctrl+A**: 全选表格数据
- **Ctrl+C**: 复制选中数据到剪贴板
  - 包含表头
  - 制表符分隔,可直接粘贴到Excel

### 4. 控制按钮

#### 编辑选中点
- **功能**: 编辑当前在可视化图中选中的点
- **快捷方式**: 双击可视化图中的点

#### 删除选中点
- **功能**: 删除当前在可视化图中选中的点
- **快捷方式**: 右键菜单 → 删除选中点

#### 添加点
- **功能**: 在指定坐标添加新点
- **操作**: 弹出对话框输入X、Y坐标

## 使用场景

### 场景1: 创建标准矩形Map
1. 点击"数据" → "新建Map"
2. 输入参数:
   - X方向长度: 140mm
   - Y方向长度: 140mm
   - X方向间距: 2mm
   - Y方向间距: 2mm
   - 中心点坐标: (0, 0)
3. 点击确定
4. 查看3001个点的矩形Map(71×71网格,去掉边界圆外的点)

### 场景2: 修改现有Map的个别点
1. 点击"数据" → "导入现有Map"
2. 选择CSV文件(如1.csv)
3. 在可视化图中双击需要修改的点
4. 在对话框中输入新的坐标值
5. 点击确定完成修改
6. 点击"数据" → "保存到CSV"保存修改后的Map

### 场景3: 删除Map中的异常点
1. 导入或创建Map
2. 在可视化图中单击要删除的点(红色高亮)
3. 点击"删除选中点"按钮
4. 保存修改后的Map

### 场景4: 导出数据到Excel分析
1. 在表格中点击(Ctrl+A)全选数据
2. 按Ctrl+C复制数据
3. 在Excel中按Ctrl+V粘贴
4. 数据包含表头,格式正确,可直接分析

## 注意事项

1. **坐标系统**: 使用笛卡尔坐标系,X轴向右为正,Y轴向上为正
2. **浮点精度**: CSV输出默认保留6位小数,自动去除末尾的0
3. **内存占用**: 大型Map(超过10000点)可能消耗较多内存,建议定期保存
4. **备份建议**: 修改重要Map前建议先备份原始文件
